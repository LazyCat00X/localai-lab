Hardware Requirements by Model Size

Before you choose a model, understand what your hardware can handle. The key metric is how much VRAM your GPU has — or system RAM if using CPU only.

Model SizeQuantizationMemory RequiredExpected Speed (GPU)Expected Speed (CPU)
3B (e.g., Llama 3.2 3B)Q4_K_M~2.5 GB100+ tok/s15-25 tok/s
7-8B (e.g., Mistral, Llama 3.1 8B)Q4_K_M~5.5 GB60-80 tok/s5-10 tok/s
13B (e.g., Llama 2 13B)Q4_K_M~8 GB30-45 tok/s2-4 tok/s
30-34B (e.g., Yi 34B)Q4_K_M~20 GB15-25 tok/s1-2 tok/s
70B (e.g., Llama 3.1 70B)Q4_K_M~40 GB5-15 tok/sToo slow
120B+ (e.g., Qwen 2.5 72B)Q3_K_M~45 GB3-8 tok/sNot feasible

Memory Rule of Thumb: Multiply the parameter count (in billions) by 0.6 to get approximate GB required at Q4_K_M quantization. At Q8, multiply by 1.2. A 7B model needs 7 × 0.6 ≈ 4.2 GB at Q4.

Understanding Quantization

Quantization reduces model precision to save memory and speed up inference. The tradeoff is a small quality loss.

Quantization Levels Compared

LevelBits per WeightQuality vs FP16SpeedUse Case
Q2_K2.6~5% lossFastestLow-memory machines, large models
Q3_K_M3.4~3% lossVery FastGood balance for 30B+ models
Q4_K_M4.5~1% lossFastRecommended default for most users
Q5_K_M5.2~0.5% lossModerateWhen quality matters more than speed
Q6_K6.6~0.2% lossSlowerHigh-quality requirement
Q8_08.3~0.1% lossSlowestMaximum quality, have memory to spare

For most users, Q4_K_M is the sweet spot. It cuts memory by 4× compared to FP16 while retaining 99%+ of the quality.

GPU Acceleration Setup

NVIDIA (CUDA)

# Check CUDA availability
nvidia-smi

# Install CUDA toolkit (Ubuntu/Debian)
sudo apt install nvidia-cuda-toolkit

# Verify
nvcc --version

# For Ollama — GPU is automatic
curl -fsSL https://ollama.com/install.sh | sh

# For llama.cpp with CUDA
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release -j$(nproc)

# For vLLM
pip install vllm
# vLLM auto-detects CUDA GPUs

AMD (ROCm)

# Ollama with ROCm
curl -fsSL https://ollama.com/install.sh | OLLAMA_ROCM=1 sh

# llama.cpp with ROCm
cmake -B build -DGGML_HIPBLAS=ON
cmake --build build --config Release -j$(nproc)
