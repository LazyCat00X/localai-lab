What Is Open WebUI?

Open WebUI is a self-hosted, feature-rich chat interface designed to work seamlessly with Ollama and other OpenAI-compatible backends. Think of it as your own local ChatGPT — a polished web UI that runs entirely on your hardware, with no data leaving your network. It supports markdown rendering, syntax-highlighted code blocks, conversation branching, RAG (Retrieval-Augmented Generation), multi-model conversations, voice input, and a full administrative dashboard for managing users, models, and permissions.

Why Open WebUI? Unlike hosted solutions, Open WebUI keeps your conversations and documents completely private. You control the backend models, the storage, and who has access. It is the most popular open-source frontend for Ollama, with over 50,000 GitHub stars and an active community.

Open WebUI replaces the need for standalone solutions like Ollama Web UI, LocalAI WebUI, or proprietary services. It provides a consistent experience whether you are running a single model on a laptop or a multi-user deployment on a server with dozens of models.

Prerequisites

Before installing Open WebUI, make sure your system meets these requirements:

  • Docker (recommended) — Install Docker Engine (Linux) or Docker Desktop (macOS/Windows).
  • Ollama — A running instance of Ollama with at least one model pulled (e.g. llama3.2, mistral, or phi3).
  • Hardware — 2 GB RAM minimum (8 GB+ recommended for multi-user use). At least 10 GB free disk space for models and document storage.
  • Port availability — Port 3000 (default Open WebUI port) must be free, or you will need to map a different port.

If you do not already have Ollama installed, here is the quick start:

# Install Ollama (Linux/macOS)
curl -fsSL https://ollama.ai/install.sh | sh

# Pull a model
ollama pull llama3.2

# Verify it is running
ollama list

Installation Methods

There are three main ways to install Open WebUI. The Docker method is by far the simplest and most reliable for most users.

Method 1: Docker (Recommended)

Run Open WebUI with a single command. Docker handles all dependencies, Python virtual environments, and persistent storage automatically.

docker run -d \
  --name open-webui \
  --restart unless-stopped \
  -p 3000:8080 \
  -v open-webui-data:/app/backend/data \
  --network=host \
  ghcr.io/open-webui/open-webui:main
Note on --network=host: This flag allows the container to reach Ollama running on localhost:11434. If you prefer bridge networking, use -e OLLAMA_BASE_URL=http://host.docker.internal:11434 instead (macOS/Windows) or your host IP (Linux).

Once started, open http://localhost:3000 in your browser. The first user you create automatically becomes the admin.

Method 2: pip Install (Python)

If you prefer to run Open WebUI without Docker, you can install it directly via pip. This requires Python 3.11 or later.

# Create a virtual environment (recommended)
python -m venv open-webui-env
source open-webui-env/bin/activate

# Install Open WebUI
pip install open-webui

# Start the server
open-webui serve

By default, the server listens on 0.0.0.0:8080. You can change the port with the --port flag. This method gives you more control over the process and integrates easily with systemd or supervisor for production use.

Method 3: Manual Setup from Source

Advanced users can clone the repository and run the frontend and backend separately. This is useful for development or customization.

git clone https://github.com/open-webui/open-webui.git
cd open-webui

# Backend
cd backend
pip install -r requirements.txt
python main.py &

# Frontend (in a separate terminal)
cd ../frontend
npm install
npm run dev

The manual approach is best if you plan to modify the UI, contribute to the project, or integrate custom authentication backends.

First Run & Connecting to Ollama

When you first load Open WebUI in your browser, you will see a login/signup screen. Register your first account — this user will be granted admin privileges automatically.

Connecting to Ollama

By default, Open WebUI looks for Ollama at http://localhost:11434. If Ollama is running on a different host or port, navigate to Admin Panel → Settings → Connections and update the Ollama Base URL field. For example:

  • Remote server: http://192.168.1.100:11434
  • Docker host (Linux with bridge): http://172.17.0.1:11434
  • Docker host (macOS/Windows): http://host.docker.internal:11434

Selecting Models

Once connected, Open WebUI automatically fetches the list of available models from Ollama. You will see them in the model selector dropdown at the top of the chat interface. You can also pull new models directly from the Open WebUI admin panel without touching the terminal — just enter the model name (e.g. mistral:7b) and click Pull.

The Chat Interface

The main chat area supports:

  • Markdown rendering — Headers, lists, tables, bold, italic, and links all render inline.
  • Code highlighting — Code blocks with language tags (e.g. ```python) are highlighted using Prism.js.
  • Conversation management — Rename, archive, delete, or export conversations from the sidebar. Conversations are searchable.
  • Multi-model chats — Tag multiple models in a single conversation and switch between them mid-chat to compare responses.
  • Voice input — Click the microphone icon to speak your prompts (requires browser microphone access).

RAG Integration

One of Open WebUI's most powerful features is built-in Retrieval-Augmented Generation. You can upload documents (PDF, TXT, Markdown, CSV, DOCX, and more) and ask questions about their content. The system uses embeddings to find relevant chunks and injects them into the model's context.

How to Use RAG

  1. Open a chat conversation.
  2. Click the paperclip / attach icon next to the message input.
  3. Upload one or more documents (up to 100 MB total per upload).
  4. Type your question about the document contents and press Enter.
  5. Open WebUI splits the document into chunks, generates embeddings, and retrieves the most relevant sections to augment the model's response.

RAG Configuration

You can fine-tune RAG behavior in Admin Panel → Settings → Documents:

Setting Default Description
Chunk Size 1000 Number of characters per document chunk. Smaller chunks increase precision but require more embedding calls.
Chunk Overlap 200 Characters of overlap between consecutive chunks to preserve context at boundaries.
Top K Results 3 Number of chunks to retrieve and include in the prompt for each query.
Embedding Model sentence-transformers/all-MiniLM-L6-v2 Model used to generate embeddings. Can be changed to any Hugging Face sentence-transformers model.
Similarity Threshold 0.0 Minimum cosine similarity score for a chunk to be included. Set higher (e.g. 0.5) to filter out weak matches.
Performance tip: For large document collections, switch to a dedicated embedding service like Ollama's nomic-embed-text model or configure an external OpenAI-compatible embeddings API. This reduces memory usage on the Open WebUI server.

RAG makes Open WebUI an excellent tool for querying internal documentation, research papers, legal contracts, or any collection of private documents. All processing happens locally — no data ever leaves your network.

User Management

Open WebUI includes a full administrative dashboard for managing users, roles, and permissions. Navigate to Admin Panel → Users to access these features.

User Roles

There are three built-in roles:

  • Admin — Full access to all settings, models, user management, and system configuration.
  • User — Can chat, upload documents, manage their own conversations, and use RAG.
  • Viewer — Read-only access. Can view conversations and documents but cannot send messages or upload files.

Permission Groups

You can create custom permission groups to control which models, tools, and features each group can access. For example, you might create a research group with access to large reasoning models and a general group limited to small, fast models. Groups can be assigned during user creation or edited later.

Email & OAuth

Open WebUI supports email-based registration (with SMTP configuration) and OAuth providers including Google, GitHub, Microsoft, and generic OpenID Connect. Configure these in Admin Panel → Settings → Authentication. For self-hosted teams, OAuth integration means users can sign in with their existing organizational accounts.

Customization

Open WebUI offers extensive customization options to tailor the experience to your needs.

Themes

Switch between light, dark, and system-follow themes from the user settings menu. You can also set a custom accent color and toggle between compact and comfortable spacing. For power users, custom CSS can be injected via the admin panel to override any visual element.

System Prompts

Define default system prompts that set the tone and behavior of models. For example, you can create a "Support Agent" persona that always responds politely and asks clarifying questions. System prompts can be set globally (affecting all users) or per-user by admins.

Model Parameters

Each model's parameters — temperature, top-p, top-k, max tokens, frequency penalty, presence penalty, and stop sequences — are fully configurable from the chat interface. Click the gear icon next to the model selector to adjust these on the fly. Saved presets let you quickly switch between "creative" (high temperature) and "precise" (low temperature) configurations.

Tools & Plugins

Open WebUI supports a growing ecosystem of tools and function plugins. These include web search, code execution (sandboxed), image generation, calculator, and custom API integrations. Tools can be enabled, disabled, and configured from the admin panel. Users can also create their own custom function tools using JavaScript directly in the UI.

Production Deployment

For a production deployment serving multiple users, you will want persistent storage, proper networking, a reverse proxy, and HTTPS.

Docker Compose (Recommended for Production)

Create a docker-compose.yml file for a complete stack with Open WebUI, Ollama, and persistent volumes:

version: "3.8"
services:
  ollama:
    image: ollama/ollama:latest
    restart: unless-stopped
    volumes:
      - ollama-data:/root/.ollama
    ports:
      - "11434:11434"

  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    restart: unless-stopped
    ports:
      - "3000:8080"
    volumes:
      - open-webui-data:/app/backend/data
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
      - WEBUI_SECRET_KEY=${WEBUI_SECRET_KEY}
      - WEBUI_NAME=LocalAI Lab
      - ENABLE_SIGNUP=false
    depends_on:
      - ollama

volumes:
  ollama-data:
  open-webui-data:

Set a strong WEBUI_SECRET_KEY environment variable. This is used to encrypt session tokens and sensitive configuration values. Generate one with openssl rand -hex 32.

Reverse Proxy with Nginx

Place Nginx in front of Open WebUI to handle SSL termination, rate limiting, and domain routing:

server {
    listen 80;
    server_name chat.localailab.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name chat.localailab.com;

    ssl_certificate /etc/letsencrypt/live/chat.localailab.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/chat.localailab.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    location /ws {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }
}

HTTPS with Let's Encrypt

Obtain free SSL certificates using Certbot:

# Install Certbot (Ubuntu/Debian)
sudo apt update && sudo apt install -y certbot python3-certbot-nginx

# Obtain a certificate
sudo certbot --nginx -d chat.localailab.com

# Verify auto-renewal
sudo certbot renew --dry-run

Important Environment Variables

Variable Default Purpose
WEBUI_SECRET_KEY auto-generated Encryption key for sessions and secrets. Set a fixed value for persistence across restarts.
WEBUI_NAME "Open WebUI" Display name shown in the browser tab and header.
ENABLE_SIGNUP true Set to false to disable public registration after your admin account is created.
OLLAMA_BASE_URL http://localhost:11434 URL of the Ollama server.
DEFAULT_MODELS null Comma-separated list of models to show by default (e.g. llama3.2,mistral).

Troubleshooting

Cannot Connect to Ollama

If Open WebUI shows "Connection refused" when trying to reach Ollama:

  • Verify Ollama is running: systemctl status ollama or ollama list.
  • Check that Ollama is listening on all interfaces if running in a separate container: OLLAMA_HOST=0.0.0.0 ollama serve.
  • When using Docker, ensure --network=host or the correct OLLAMA_BASE_URL is set.

White Screen on Load

A blank white screen usually indicates a frontend build issue or a JavaScript error. Open your browser's developer console (F12) and look for errors. Common causes include:

  • Corrupted browser cache — try a hard refresh (Ctrl+Shift+R) or clear site data.
  • Outdated container — pull the latest image: docker pull ghcr.io/open-webui/open-webui:main.
  • Port conflict — another service may be using port 3000 or the mapped port.

Document Upload Fails

If document uploads for RAG fail silently:

  • Check available disk space — Open WebUI stores uploaded documents in its data volume.
  • Verify the file type is supported (PDF, TXT, MD, CSV, DOCX, EPUB, HTML).
  • Check the container logs: docker logs open-webui --tail 50.

High Memory Usage

Open WebUI caches embeddings and conversation history in memory. For long-running deployments:

  • Set MAX_CONVERSATION_HISTORY to a reasonable limit (e.g. 1000).
  • Use an external vector database (Chroma, Qdrant) instead of the default in-memory store for production RAG workloads.
  • Monitor memory usage with docker stats open-webui and adjust the container's memory limit in Docker Compose.

Next Steps

Now that Open WebUI is up and running, here are some things you can explore:

  • Experiment with different Ollama models and compare their outputs side-by-side.
  • Set up a shared knowledge base by uploading company documentation and enabling RAG for all users.
  • Configure OAuth to let your team sign in with their existing Google or GitHub accounts.
  • Write custom function tools to integrate with your internal APIs.
  • Explore the Open WebUI GitHub repository for the latest features and community plugins.

Open WebUI transforms a bare Ollama installation into a polished, production-ready AI assistant platform. Whether you are running it on a single Raspberry Pi or a multi-GPU server, it scales with your needs and keeps your data entirely under your control.
