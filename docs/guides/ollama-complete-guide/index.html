1. What Is Ollama?

Ollama is the most popular open-source tool for running large language models locally. It wraps llama.cpp (and on Apple Silicon, MLX) behind a simple CLI and REST API, making it trivial to download, run, and interact with hundreds of models — Gemma, LLaMA, Qwen, DeepSeek, Mistral, Phi, and more — on consumer hardware.

As of June 2026, Ollama has 175,000+ GitHub stars, over 5,400 commits, and 227 releases. Version 0.30 (released June 2026) brought Vulkan GPU acceleration by default, up to 20% faster NVIDIA throughput, native GGUF file support, and a unified MLX + GGUF engine architecture.

Key Features: Ollama supports tool calling for coding agents (Claude Code, Codex, OpenCode, Hermes Agent), image generation (experimental, macOS), web search API, an Anthropic-compatible API mode, ollama launch for one-command integration setup, and intelligent model scheduling for multi-GPU systems.

2. Installation

Linux (x86_64 / ARM64)

The recommended install method is a single curl pipe:

$ curl -fsSL https://ollama.com/install.sh | sh

This script detects your OS, installs system dependencies, and sets up Ollama as a systemd service. After installation, verify it's running:

$ systemctl --user status ollama
● ollama.service - Ollama Service
     Loaded: loaded
     Active: active (running)

$ ollama --version
ollama version 0.30.11

macOS (Apple Silicon)

Download the macOS app from ollama.com or use Homebrew:

$ brew install ollama
$ ollama serve

On Apple Silicon, Ollama now uses its MLX engine by default for optimal performance. Models output higher quality responses, respond faster, and use less memory compared to the previous llama.cpp-based engine on macOS.

Windows

$ irm https://ollama.com/install.ps1 | iex

Or download the installer from ollama.com. Windows builds include Vulkan GPU acceleration out of the box.

Docker

$ docker run -d --gpus all -v ollama:/root/.ollama -p 11434:11434 \
    --name ollama ollama/ollama
GPU Note: On Linux, the install script handles NVIDIA GPU setup automatically if it detects the CUDA toolkit and NVIDIA drivers. For AMD GPUs, ensure ROCm is installed — Ollama's Vulkan backend works with AMD, Intel, and NVIDIA hardware without vendor-specific libraries.

5. REST API

Ollama exposes a REST API on localhost:11434. This is how applications like Open WebUI, Continue, and coding agents connect to Ollama.

Chat Completion

$ curl http://localhost:11434/api/chat -d '{
  "model": "gemma4",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "What is the capital of France?"}
  ],
  "stream": false
}'

Response:

{
  "model": "gemma4",
  "message": {
    "role": "assistant",
    "content": "The capital of France is Paris."
  },
  "done": true,
  "total_duration": 1854321000,
  "load_duration": 102340000,
  "tokens_evaluated": 12,
  "tokens_per_second": 23.4
}

Generate (Simple Completion)

$ curl http://localhost:11434/api/generate -d '{
  "model": "gemma4",
  "prompt": "Write a haiku about Linux"
}'

List Local Models

$ curl http://localhost:11434/api/tags

Check Model Loading Status

$ curl http://localhost:11434/api/ps

OpenAI-Compatible Endpoint

Ollama also serves an OpenAI-compatible endpoint at /v1/, making it a drop-in replacement for OpenAI in any application that supports custom endpoints:

$ curl http://localhost:11434/v1/chat/completions -d '{
  "model": "gemma4",
  "messages": [{"role": "user", "content": "Hello!"}]
}'

Set OPENAI_BASE_URL=http://localhost:11434/v1 and OPENAI_API_KEY=ollama in your application to route traffic through Ollama.

6. Python SDK

Ollama provides a first-party Python package that wraps the REST API:

$ pip install ollama

from ollama import chat

response = chat(model='gemma4', messages=[
    {'role': 'user', 'content': 'Why is the sky blue?'},
])
print(response.message.content)

Streaming Responses

from ollama import chat

stream = chat(model='gemma4', messages=[
    {'role': 'user', 'content': 'Tell me a story'},
], stream=True)

for chunk in stream:
    print(chunk.message.content, end='', flush=True)

Tool Calling with Python

Ollama supports tool/function calling in the Python SDK:

from ollama import chat

def get_weather(city: str) -> str:
    return f"Weather in {city}: 22°C, partly cloudy"

response = chat(model='gemma4', messages=[
    {'role': 'user', 'content': 'What is the weather in Tokyo?'},
], tools=[get_weather])

print(response.message.tool_calls)

7. Custom Modelfiles

Modelfiles let you customize model behavior — set system prompts, adjust temperature, configure context length, and import GGUF files from Hugging Face.

Basic Modelfile

FROM gemma4

# System prompt
SYSTEM "You are an expert Python developer. Always include code examples."

# Override parameters
PARAMETER temperature 0.3
PARAMETER top_p 0.9
PARAMETER num_ctx 8192

Create and run:

$ ollama create my-coder -f ./Modelfile
$ ollama run my-coder

Importing a GGUF Model from Hugging Face

As of Ollama 0.30, you can import GGUF files directly:

# Download a GGUF file first
$ wget https://huggingface.co/bartowski/LFM-40B-GGUF/resolve/main/LFM-40B-Q4_K_M.gguf

# Create Modelfile
$ cat > Modelfile << 'EOF'
FROM ./LFM-40B-Q4_K_M.gguf
PARAMETER num_ctx 4096
EOF

$ ollama create lfm-40b -f Modelfile
$ ollama run lfm-40b

Available Parameters

ParameterDefaultDescription
temperature0.7Controls randomness. Lower = more deterministic.
top_p0.9Nucleus sampling threshold.
num_ctx2048Context window size in tokens.
repeat_penalty1.1Penalize repeated tokens.
num_predict-1Max tokens to generate (-1 = unlimited).
stop[]Stop sequences (e.g., ["\n", "User:"]).
mirostat0Mirostat sampling mode (0=off, 1=on).

8. Advanced Features

ollama launch

Introduced in January 2026, ollama launch sets up coding tools and AI assistants with local or cloud models in a single command:

# Launch Claude Code with your local model
$ ollama launch claude --model gemma4

# Launch Hermes Agent
$ ollama launch hermes --model gemma4

# Launch OpenClaw (multi-platform AI assistant)
$ ollama launch openclaw --model gemma4

# Launch OpenAI Codex
$ ollama launch codex --model gpt-oss:20b

This eliminates environment variable and config file setup — Ollama handles model discovery, API endpoint configuration, and tool integration automatically.

Anthropic Messages API Compatibility

Ollama now serves the Anthropic Messages API format, making it compatible with Claude Code and other Anthropic-native tools:

$ curl http://localhost:11434/anthropic/v1/messages -d '{
  "model": "gemma4",
  "max_tokens": 1024,
  "messages": [{"role": "user", "content": "Hello"}]
}'

Web Search API

Ollama provides a built-in web search API (generous free tier for individuals):

Image Generation (Experimental)

As of January 2026, Ollama can generate images locally on macOS:

POST /api/generate
{
  "model": "gemma4",
  "prompt": "Generate an image of a cat wearing a space helmet"
}
Platform support: Image generation is currently experimental and macOS-only. Windows and Linux support are in development.

Model Scheduling

Ollama's intelligent scheduler (September 2025) manages memory across multiple models, reducing OOM crashes and maximizing GPU utilization:

  • Memory-aware loading: Ollama unloads idle models when GPU memory is needed
  • Multi-GPU scheduling: Distributes model layers across available GPUs
  • Concurrent model serving: Multiple models can remain loaded within memory budget

Check loaded models with:

$ ollama ps
NAME            ID              SIZE    PROCESSOR
gemma4:latest   abc12345        16 GB   100% GPU
mistral:latest  def67890        4.1 GB  100% GPU

Tool Calling for Coding Agents

Ollama exposes model tool-calling capabilities to connected applications. Verify tool support:

$ ollama show gemma4 | grep -i tool
  tools: enabled

Coding agents like Claude Code, Codex CLI, Cline, and Continue can use Ollama models with full function-calling support — reading files, writing code, running shell commands, and browsing the web through the agent's tools.

10. Performance Tuning

GPU Acceleration

Ollama automatically detects and uses your GPU. Check what backend is active:

$ ollama ps
NAME            ID              SIZE    PROCESSOR
gemma4:latest   abc12345        16 GB   100% GPU

If the processor shows "CPU", GPU acceleration isn't working. Common fixes:

  • NVIDIA: Ensure CUDA drivers are installed (nvidia-smi shows your GPU)
  • AMD: Install ROCm and ensure Vulkan drivers are present
  • Intel: Install oneAPI or use the Vulkan backend

Environment Variables for Optimization

VariablePurposeExample
OLLAMA_NUM_PARALLELParallel request handlingOLLAMA_NUM_PARALLEL=4
OLLAMA_MAX_LOADED_MODELSMax models kept in memoryOLLAMA_MAX_LOADED_MODELS=2
OLLAMA_KEEP_ALIVEHow long to keep model loadedOLLAMA_KEEP_ALIVE=5m
OLLAMA_HOSTBind addressOLLAMA_HOST=0.0.0.0
OLLAMA_DEBUGEnable verbose loggingOLLAMA_DEBUG=1

Multi-GPU Setup

# Distribute model across two GPUs
$ OLLAMA_NUM_PARALLEL=2 ollama run gemma4

Ollama's scheduler distributes model layers across available GPUs automatically. Monitor utilization with nvidia-smi or ollama ps.

Context Length and Memory

Increasing context length consumes more GPU memory. A rough formula:

Memory ≈ model_size + (context_length × bytes_per_token)

# Example: Gemma4 26B Q4 with 8192 context
# 16 GB + (8192 × 4 bytes × 32 layers × ~0.5) ≈ 16 GB + 0.5 GB ≈ 16.5 GB

If you run out of memory, try a smaller quantization (:q2_k) or reduce num_ctx in your Modelfile.

Performance Tip: For production deployments, set OLLAMA_KEEP_ALIVE=0 to unload models immediately after each request (saves memory at the cost of reload time), or OLLAMA_KEEP_ALIVE=-1 to keep them permanently loaded (maximum throughput, higher memory usage).
